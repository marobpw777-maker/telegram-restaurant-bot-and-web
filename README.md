# Ресторан Вкус 2026 🚀

Профессиональный Telegram-бот для бронирования столиков, AI-поддержки гостей, веб-админки и unit economics ресторана.

![Python](https://img.shields.io/badge/python-3.11%2B-blue) ![React](https://img.shields.io/badge/react-18%2B-blue) ![License](https://img.shields.io/badge/license-MIT-green)

---

## ✨ Возможности

### 🍽️ Бронирования
- Бронирование столика через Telegram и веб-интерфейс
- Проверка доступности слотов по дате и времени
- Подтверждение и отклонение брони администратором
- Напоминания гостю за 24 часа и за 1 час до визита
- Отмена брони из напоминания
- Отслеживание no-show и CRM-метрик

### 📖 Меню и витрина ресторана
- Категории и карточки блюд
- Фото блюд, описание, состав, аллергены, цена и вес
- Отдельная винная карта
- Быстрые CTA для перехода к бронированию
- Адаптивный веб-интерфейс для гостей

### 🤵 AI-ассистент
- Цифровой консьерж для общих вопросов о ресторане
- Сомелье для подбора вина под блюда
- Fallback-ответы, если AI-ключ не задан
- Работа по контексту диалога и истории сообщений

### 👑 Админ-панель
- Вход по логину и паролю из .env
- Просмотр и обработка бронирований
- Редактирование меню и категорий
- Маркетинговые рассылки гостям, которые согласились с политикой конфиденциальности
- Dashboard с бронями, гостями, выручкой, no-show и прогнозами
- Отдельный блок unit economics с настраиваемыми параметрами

### 📊 Unit economics
- Средний чек, загрузка, no-show и брони в месяц
- Выручка, gross profit, operating profit и net profit
- Фиксированные и переменные расходы
- Break-even point
- Сценарный анализ по среднему чеку, no-show и загрузке
- Сохранение финансовых настроек в БД

### 🔒 Приватность и контроль
- Согласие на обработку персональных данных
- Защита админки через сессию
- Telegram-уведомления только там, где у гостя есть Telegram ID
- Веб-бронирования с fallback на телефонный контакт

## 🪟 Способ 1: запуск через CMD (Windows)

### Шаг 1. Откройте командную строку
Нажмите **Win + R**, введите `cmd`, нажмите **Enter**.

### Шаг 2. Перейдите в папку проекта
```bash
cd C:\path\to\your\restaurant-bot\BOTREST
```

### Шаг 3. Создайте виртуальное окружение
```bash
python -m venv venv
venv\Scripts\activate
```

### Шаг 4. Установите зависимости
```bash
python -m pip install --upgrade pip
pip install -r requirements.txt
```

### Шаг 5. Заполните .env
Скопируйте шаблон и откройте файл:
```bash
copy .env.example .env
```

**Минимально должны быть заданы:**
```
TELEGRAM_BOT_TOKEN=ваш_токен_от_BotFather
ADMIN_LOGIN=admin
ADMIN_PASSWORD=change_me_now
ADMIN_IDS=123456789
ADMIN_SESSION_SECRET=long_random_secret
WEB_APP_URL=http://localhost:5173
DATABASE_URL=sqlite+aiosqlite:///./data/restaurant.db
```

### Шаг 6. Запустите backend + Telegram-бота
```bash
python main.py
```
✅ **Готово.** Бот, API и планировщик будут запущены в одном процессе.

### 🌐 Шаг 7. Запустите веб-интерфейс и админку
Во втором окне терминала:
```bash
cd C:\path\to\your\restaurant-bot\Веб-интерфейс\app
npm install
npm run dev
```

Обычно Vite откроется на `http://localhost:5173`, но если порт занят — используйте адрес, который он покажет в терминале.

## 📈 Что нужно для production

Перед боевым запуском лучше заменить и донастроить:

- SQLite → PostgreSQL
- Локальные фоновые задачи → отдельный worker/queue слой
- Ручные ключи → secret manager
- Базовую сессию админки → более строгую auth-модель
- Development-only конфиги → production окружение
- Простой логинг → централизованный мониторинг и аудит
- Локальный веб-хостинг → HTTPS/SSL и нормальный деплой

## 🆘 Частые проблемы

### Бот не стартует
Проверьте, не занят ли порт 8000, и нет ли уже запущенного backend-процесса.

### Админка открывается пустой
Проверьте, что backend запущен, а WEB_APP_URL указывает на правильный адрес фронтенда.

### AI отвечает fallback-текстом
Это нормально, если не задан GIGACHAT_API_KEY.

### Не приходят Telegram-уведомления веб-клиенту
Если у гостя нет telegram_id, система использует телефонный fallback и уведомление админу.
